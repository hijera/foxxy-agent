# OpenAI-compatible HTTP API

The `foxxycode http` subcommand ships only when the binary is built with **`go build -tags=http`** (`make build TAGS=http`). It exposes OpenAI-shaped routes plus **`/foxxycode/*`** helpers, backed by the same session manager and ReAct agent as **`foxxycode acp`**.

The bundled SPA is included only when you also set the **`ui`** tag (for example **`make build TAGS="http ui"`** or **`go build -tags=http,ui`**). With **http** only, **`GET /`** returns **404** with a plain-text hint.

## OpenAPI and Swagger UI

Specs are regenerated on each request so they stay aligned with handlers.

- **`GET /openapi.yaml`** - OpenAPI 3.0 (YAML); **`GET /openapi.json`** mirrors it in JSON (both **`Content-Disposition: inline`**).
- **`GET /docs/`** - Swagger UI (static assets bundled in the binary, no CDN). **`GET /docs`** redirects to **`/docs/`**.

No authentication is enforced. Run behind appropriate network controls.

## Embedded web UI (**`-tags=http,ui`**)

- **`GET /`** serves **`index.html`**, **`styles.css`**, and **`app.js`** from **`external/ui/`** (`go:embed`) when **`ui`** is set at link time with **`http`**. Responses for **`/`**, **`/index.html`**, **`/app.js`**, and **`/styles.css`** include **`Cache-Control: no-cache`** so a normal reload picks up assets after you rebuild the binary (fixed URLs, no fingerprint).
- Recommended session URL pattern: **`#/s/<sessionId>`** (client-only history). For **`POST /v1/responses`** and **`POST /v1/chat/completions`**, send **`X-FoxxyCode-Session-ID`** with a **`sess_<hex>`** id that satisfies server-side validation (`internal/session/validate.go`).
- **`GET /foxxycode/sessions`**, **`PATCH /foxxycode/sessions/{id}`**, and other **`/foxxycode/sessions/{id}/...`** routes identify the session **only** by the **`{id}`** path segment (no duplicate header in OpenAPI; clients may still send **`X-FoxxyCode-Session-ID`**, it is not used to pick the resource for those URLs).
- The bundled UI calls **`POST /v1/responses`** with **`stream: true`**. If **`GET /foxxycode/sessions`** reports **`turnActive`** for the open session (for example after a full tab reload while another client or the same server process still runs the turn), the UI subscribes to **`GET /foxxycode/sessions/{id}/composer-stream`** (SSE) to replay bytes already emitted for that turn and continue receiving live chunks until **`data: [DONE]`** or an error event. Starting a new **`POST /v1/responses`** cancels that relay subscription client-side. For ReAct (**`model`** **`agent`**, **`plan`**, or **`docs`**), it sends optional **`metadata.model`** with the selected YAML **`models[].model`** **`id`** and optional **`metadata.runPlanSlug`** to implement a saved design plan (**`foxxycode.dev/runPlanSlug`** on ACP **`session/prompt`**) (from **`GET /v1/models`**, **`owned_by`** not **`foxxycode`**); default follows **`default_agent_model`**, persisted choice in **`foxxycode_llm_model`**. When the selected model exposes **`reasoning_levels`**, it also sends optional **`metadata.reasoning`** with the chosen level; default follows the model's **`reasoning_default`**, persisted choice in **`foxxycode_llm_reasoning`**. It derives **`attachments`** from **`@path`** mentions in **`input`** (files only); **`GET /foxxycode/workspace/files`** powers the **`@`** picker. SSE **default** chunks follow **`chat.completion.chunk`** deltas (**`delta.content`** for assistant text, **`delta.reasoning_content`** for streamed model reasoning so the UI can keep the **thinking** foldout separate). Named **`event:`** lines (**`tool_call`**, **`tool_call_update`**, **`plan`**, **`token_usage`**, **`available_commands`**) expose tool progress and incremental token totals between LLM rounds; when **`memory.enabled`** is **true**, the server also emits **`memory_phase`** (**`sessionUpdate`** **`memory_phase`**) and **`memory_chunk`** (**`memory_message_chunk`**) for the unified memory copilot (**one pass before the main model**, recall-or-persist), streamed model text deltas only, without adding rows to **`messages.json`**. **`GET /foxxycode/sessions/{id}/messages`** may include **`memoryTurns`** (array of persisted per-turn summaries for the bundled UI transcript). **`foxxycode_meta`** JSON follows the usual rule immediately before **`data: [DONE]`**. **Stop generation** in the composer calls **`POST /foxxycode/sessions/{id}/cancel`** with **`X-FoxxyCode-Session-ID`** matching **`{id}`**, then **`AbortSignal`** tears down the streamed **`fetch`**. That maps to ACP **`session/cancel`** for the agent turn (**`TurnCtx`** propagation for direct **`models[].model`** completions uses the same cancel hook). Cancel mid-stream still **persists** accumulated assistant **`content`** for that turn; **`GET /foxxycode/sessions/{id}/messages`** can briefly trail that write, so the SPA **merges** fetched rows with its per-session shadow or on-screen transcript (**`mergeTranscriptPreferLocalSuffix`** in **`external/ui/src/ui/chat/transcriptServerSnapshot.ts`**). More than one session may stream in parallel (**`pickStreamMutationBase`** in **`streamMutationBase.ts`** keeps SSE mutations scoped per **`X-FoxxyCode-Session-ID`**).
- On the first user message of a brand-new chat (no session in the rail yet), the UI **`POST`**s **`/foxxycode/describe`** in parallel with **`/v1/responses`**. When **`short`** arrives, the header and session list titles update **immediately**; **`PATCH /foxxycode/sessions/{id}`** still runs once headers expose the canonical session id (**retries PATCH** briefly if persistence lags describe).
- The bundled UI exposes **Settings** at **`#/settings`**. It loads **`GET /foxxycode/config/schema`** (JSON Schema for the editor) and **`GET /foxxycode/config`** (current values, including **API keys**, optional per-provider **`proxy`** URLs, and the **`gateways.telegram.token`** bot token), then **`POST /foxxycode/config/validate`** and **`PUT /foxxycode/config`** on save. The **`gateways`** block round-trips through this API, so editing settings in the UI no longer drops the Telegram configuration. There is no separate auth on these routes - expose **`foxxycode http`** only on trusted networks.

| GET | `/` | Embedded web UI (**`-tags=http,ui`**) or **404** with a hint (**`http` only**). **`Cache-Control: no-cache`** on **`/`**, **`/index.html`**, **`/app.js`**, **`/styles.css`** when the UI module is linked. |

| GET | `/v1/models` | Merged list: **`agent`**, **`plan`**, **`docs`** (each **`owned_by`**: **`foxxycode`**), then every YAML **`models[].model`** row (**`id`** is the selector; **`owned_by`** is the provider prefix). Ordering is **`agent`**, **`plan`**, **`docs`**, then **`models`** in configuration order (same ordering the server emits). Use any returned **`id`** as **`model`** on POST. The JSON object also includes optional **`default_agent_model`**, the configured **`agent.model`** (**`models[].model`**), when set; the bundled UI uses it as the default **`metadata.model`** for ReAct (**`agent`** / **`plan`** / **`docs`**) requests. Each **`models[].model`** row also carries **`max_context_tokens`** (UI hint; 0 or absent for foxxycode-session profiles) and **`multimodal`** (**`true`** when **`models[].multimodal: true`** in YAML; the bundled UI shows a file attachment button in the composer when the selected model has **`multimodal: true`**). Reasoning models also carry **`reasoning_levels`** (array, e.g. **`["minimal","low","medium","high"]`**; absent for non-reasoning models) and optional **`reasoning_default`** (level pre-selected for new chats); the bundled UI shows a reasoning-level selector in the composer when **`reasoning_levels`** is non-empty. |
| POST | `/v1/chat/completions` | **`stream`**, **`messages`** (last **`user`**). **`409`** when another **agent**, **plan**, or **docs** turn already holds the session turn lock (exclusive per persisted bundle). |
| POST | **`/v1/responses`** | **`model`**, **`input`**, optional **`stream`**, optional **`attachments`** ( **`path`** workspace-relative under session cwd, **`agent`** / **`plan`** / **`docs`** only; server rejects traversal, oversized, non UTF-8, and folder **`path`**), optional **`inline_files`** (array of **`{name, data_url}`**; **`data_url`** is a `data:<mime>;base64,...` URI; works for all modes — direct YAML model and **`agent`** / **`plan`** / **`docs`**). For direct YAML model: each entry becomes an image content part on the user message so multimodal models receive the file inline. **`input`** remains the full composer text including **`@path`** echoes. Keeps history between turns when using headers. **`409`** when another **agent**, **plan**, or **docs** turn already holds the session turn lock. |
| GET | `/v1/responses/{id}` | Session metadata snapshot. |
| GET | **`/foxxycode/sessions`** | Pagination via **`limit`/`cursor`** (cursor is numeric offset token). Optional **`q`** substring filter (**case-insensitive**) over **`session.json` title OR the **first persisted `user`** message **`content`** in `messages.json` order (`system`, `assistant`, `tool`, etc. before that first `user` are ignored). **`q` is not full-text search** over the transcript. Pagination applies **after** filtering. Rows sort by **`session.json` `updatedAt`** (newest first), then **`id`** for stable ties; **`updatedAt`** moves forward on persistence (new turns, title pin, etc.), not when the server only loads a bundle to serve **GET** requests. **`include_scheduler=true`** includes session bundles created for **scheduler runs** (they set **`schedulerRun`** in **`session.json`** and are **omitted** from the default composer list). **`include_activity=true`** adds **`turnActive`**, **`activitySeq`**, **`readActivitySeq`**, and **`unreadComplete`** to each row for multi-tab UI. |
| POST | **`/foxxycode/describe`** | JSON **`{"text"}`** to get a short description phrase. |
| POST | **`/foxxycode/enhance-prompt`** | JSON **`{"text"}`** — rewrites the user's draft prompt into a clearer, more specific one via a single direct LLM completion (no tools). The draft is treated only as source text to improve, never as a request to answer. JSON **`{"object":"foxxycode.enhance_prompt","text"}`**; **`400`** on blank text, **`502`**/**`503`** on LLM error/unavailability. The composer wand button (**`data-testid="composer-enhance-btn"`**) calls this and replaces the draft; **Ctrl+Z**/**⌘Z** restores the pre-enhance text. |
| GET | **`/foxxycode/slash-commands`** | Paginated slash commands derived from configured skills (**`page`**, **`page_size`** required). Optional **`prefix`** filters by **`name`** (case-insensitive). Optional **`X-FoxxyCode-Session-ID`** resolves **`${CWD}`** expansion for **`skills.dirs`** like other session-scoped lookups. JSON **`items[].name`** and **`items[].description`**, **`total`**, **`has_more`**. |
| GET | **`/foxxycode/skills`** | Lists all skills found in configured **`skills.dirs`**. JSON **`{"object":"foxxycode.skills_list","items":[...]}`**; each item has **`name`**, **`description`**, **`file_path`**, **`enabled`**. Uses same directory scan and last-wins deduplication as the agent loader. |
| POST | **`/foxxycode/skills/{name}/enable`** | Removes **`{name}`** from the disabled list in **`ManagedDir`** (`~/.foxxycode/skills/.disabled`). **`200`** **`{"ok":true}`**; **`400`** when **`name`** contains path separators or is blank. Invalidates the slash-command cache so the next **`/foxxycode/slash-commands`** call reflects the change. |
| POST | **`/foxxycode/skills/{name}/disable`** | Adds **`{name}`** to the disabled list. Same response codes and cache invalidation as **`enable`**. |
| GET | **`/foxxycode/workspace/files`** | Paginated workspace listing under session **cwd** (**`page`**, **`page_size`** required). Non-blank **case-insensitive** substring **`prefix`** on **`path_rel`**; omit or blank **`prefix`** yields empty **`items`**. **`dirs=true`** adds directory rows (**`kind`** **`dir`**, **`path_rel`** trailing **`/`**). JSON **`items[].name`**, **`items[].path_rel`**, **`items[].kind`** (**`file`** or **`dir`**), **`total`**, **`has_more`**. |
| GET | **`/foxxycode/onboarding/status`** | First-run onboarding probe for the embedded UI. Returns **`first_run`**, **`has_providers`**, **`has_models`**, **`has_agent_model`**, **`missing_api_keys`**, and optional **`suggested_defaults`**. The SPA opens the provider picker modal when onboarding is incomplete. |
| GET | **`/foxxycode/project`** | Current project folder. JSON **`{"object":"foxxycode.project","path","source","native_picker"}`**; **`source`** is **`project`** when a project was opened explicitly, **`default`** when falling back to the process cwd (**`-cwd`** / **`FOXXYCODE_CWD`** / startup dir). **`native_picker`** tells the SPA whether **`pick-folder`** can open the native OS dialog (desktop app). |
| PUT | **`/foxxycode/project`** | Body **`{"path"}`** — opens a project folder. The path must name an existing directory (**`400`** otherwise; **`503`** when the project store is unavailable). **New** sessions created afterwards use it as their working directory; existing sessions keep their own cwd. Also bumps the persisted recent list (`~/.foxxycode/projects.json`, capped at 15). |
| GET | **`/foxxycode/projects/recent`** | Recently opened projects, most recent first. JSON **`{"object":"list","data":[{"path","name","last_opened_at","exists"}]}`**; entries whose folder vanished keep **`exists:false`** so the UI can dim them. |
| POST | **`/foxxycode/project/pick-folder`** | Desktop only: opens the native OS folder dialog owned by the app window and returns **`{"object":"foxxycode.project_pick","cancelled","path"}`**. Does **not** change the current project — confirm via **PUT** **`/foxxycode/project`**. **`501`** outside the desktop app, **`409`** while another dialog is already open. |
| GET | **`/foxxycode/config/schema`** | JSON Schema (draft 2020-12) for the settings editor. Describes the same logical fields as **`ConfigJSON`** with **`description`**, **`default`**, **`pattern`** on **`providers[].name`**, optional **`x-foxxycode-*`** hints (for example **`x-foxxycode-provider-api-key-env-placeholder`** on **`providers[].api_key`** for dynamic placeholders), and **`x-foxxycode-property-order`** for section order. **`httpserver`** is omitted (listen bind stays on the **`foxxycode http`** CLI). **`GET`** / **`PUT`** **`/foxxycode/config`** still return the full document including **`httpserver`** when present in YAML. |
| GET | **`/foxxycode/config`** | Current process configuration as JSON (same logical fields as **`config.yaml`**, including secrets). |
| POST | **`/foxxycode/config/validate`** | Body is **`ConfigJSON`**. **`200`** with **`{"ok":true}`** when valid, **`400`** with **`{"ok":false,"error":"..."}`** otherwise. |
| PUT | **`/foxxycode/config`** | Body is **`ConfigJSON`**. Validates, writes **`config.yaml`** atomically, reloads in-process config for HTTP handlers and new agent turns. **`200`** **`{"ok":true}`**; **`400`** on validation failure; **`500`** if the file was written but reload failed (server restores **`config.yaml.bak`** over the primary path when possible). Existing sessions keep their current **MCP** client connections until you open a new session. |
| GET | **`/foxxycode/providers/{name}/models`** | Fetches the model list advertised by the named provider's server (openai: **`GET {api_base}/models`**; anthropic: **`GET {api_base}/v1/models`**). Provider is resolved from the saved config, so its credentials (**`api_key`** / **`api_key_command`** / **`NAME_API_KEY`**) and **`proxy`** apply server-side. **`200`** **`{"ok":true,"models":[{"id","name"}]}`** on success; **`200`** **`{"ok":false,"error":"...","models":[]}`** when the upstream call fails (UI falls back to manual entry); **`404`** for an unknown provider name. |
| POST | **`/foxxycode/providers/models-probe`** | Model list for a provider **not saved in the config yet** (onboarding): body **`{"type","api_base","api_key","proxy"?}`** carries the credentials instead of resolving them by name (**`type`** is **`openai`** or **`anthropic`**; empty **`api_base`** uses the type default). Response shape matches **`GET /foxxycode/providers/{name}/models`**: **`200`** **`{"ok":true,"models":[...]}`** or **`200`** **`{"ok":false,"error":"...","models":[]}`** on upstream failure; **`400`** for a malformed body or unsupported **`type`**. |
| GET | **`/foxxycode/sessions/{id}/activity`** | Composer activity snapshot **`turnActive`** (exclusive turn lock), **`activitySeq`**, **`readActivitySeq`**, **`unreadComplete`**. |
| GET | **`/foxxycode/sessions/{id}/tool-calls`** | Tool calls timeline; each **`resultPreview`** is capped at **19** content lines plus a final **`...`** row when truncated (20 visible preview rows total; see **`resultPreviewTruncated`**, **`resultTotalLines`**). |
| GET | **`/foxxycode/sessions/{id}/tool-calls/{toolCallId}`** | Always full persisted **`result`** plus **`meta`** and **`args`**. The bundled SPA requests this when the user activates **Load more results** after the streamed or list preview was truncated (not when merely expanding the disclosure). |
| GET | **`/foxxycode/sessions/{id}/stats`** | Session stats: token usage totals (and optional per turn list). |
| GET | **`/foxxycode/sessions/{id}/messages`** | OpenAI-shaped **`messages`**, optional **`memoryTurns`**, **`uiLog`**. After **`POST .../cancel`**, the last assistant row may appear on disk one persistence step later than the in-flight stream; clients that already showed partial text should merge with local state (bundled UI uses **`mergeTranscriptPreferLocalSuffix`** in **`external/ui/src/ui/chat/transcriptServerSnapshot.ts`**). |
| GET | **`/foxxycode/sessions/{id}/composer-stream`** | **`text/event-stream`** relay of the live **agent**/**plan**/**docs** turn (same SSE as **`POST /v1/responses`** with **`stream: true`**). Replays bytes captured so far, then live tail until the turn ends. SSE comment **`: composer stream pending`** while waiting for a relay (about 30s max), then **`event: error`** if none appears. |
| POST | **`/foxxycode/sessions/{id}/permission`** | Resolves a tool permission prompt while a streamed ReAct turn is blocked. Body **`{"toolCallId","optionId"}`** where **`toolCallId`** matches **`event: permission`** on the composer SSE (**`toolCall.toolCallId`**). **`optionId`** is **`allow`**, **`allow_always`**, or **`reject`** (or send **`outcome`** **`allow`** / **`cancelled`**). **`204`** when accepted. **`404`** when nothing is waiting. Optional **`X-FoxxyCode-Session-ID`** must match **`{id}`** when set. |
| GET | **`/foxxycode/ide/events`** | **`text/event-stream`** side channel for native editor clients (e.g. the IntelliJ plugin) to render inline diffs. Process-global (one per workspace). Emits **`event: edit_proposed`** when a **`write`**/**`edit`**/**`apply_patch`** tool is awaiting permission (gated mode), and **`event: edit_applied`** after a successful write. Each **`data`** is **`{"type","toolCallId","sessionId","path","before","after"}`** with an absolute **`path`** and full file **`before`**/**`after`**. Resolve a gated edit via **`POST /foxxycode/sessions/{id}/permission`**. |
| POST | **`/foxxycode/sessions/{id}/question`** | Answers the interactive **`question`** tool while a streamed ReAct turn is blocked. Body **`{"requestId","answers"}`** where **`answers`** is an array of string arrays (one row per question, entries are option labels or custom text). **`requestId`** must match **`event: question`** on the composer SSE. **`204`** when accepted. **`404`** when nothing is waiting. Optional **`X-FoxxyCode-Session-ID`** must match **`{id}`** when set. |
| PATCH | **`/foxxycode/sessions/{id}`** | **`{"title"}`** pins **`session.json.titlePinned`**. **`{"markActivityRead":true}`** advances **`readActivitySeq`** to the current **`activitySeq`** (clears unread dot in composer UI) **without** changing **`session.json.updatedAt`**, so the session does not jump in the history list. At least one field is required. |
| POST | **`/foxxycode/sessions/{id}/cancel`** | Best-effort cancel of an in-flight ReAct turn or streamed direct completion (**ACP `session/cancel`**). For persisted bundles, also writes a cross-process cancel marker so another **`foxxycode`** process holding the turn can stop cooperatively. When the LLM had already streamed assistant tokens, the agent **persists** that partial **`assistant`** row before ending the turn (**`StopReasonCancelled`**). Optional **`X-FoxxyCode-Session-ID`** must match **`{id}`** when present. **`404`** when no live or persisted bundle exists under that **`id`** (session store required for cold loads). |
| DELETE | **`/foxxycode/sessions/{id}`** | Removes the entire persisted session directory (including `tool_calls/` and `stats.json`) plus in-memory MCP clients. |
| GET/PUT | **`/foxxycode/sessions/{id}/plan`** | Read or overwrite todo **`entries`** (ACP shape). |
| POST | **`/foxxycode/sessions/{id}/plan/archive`** | Archives active todos like **`foxxycode_todo_plan_archive`**. |
| GET/POST | **`/foxxycode/sessions/{id}/plans`** | List or create design plan files **`plans/<slug>.plan.md`** (bundled UI; not core ACP). |
| GET/PUT/DELETE | **`/foxxycode/sessions/{id}/plans/{slug}`** | Read, autosave, or discard a design plan file. |
| PATCH | **`/foxxycode/sessions/{id}/plans/{slug}`** | Body **`{"runPlan":true}`** runs **`RunPlan`** (sync). Prefer **`POST /v1/responses`** with **`metadata.runPlanSlug`** for streaming. |

After **`PUT`** **`/foxxycode/config`**, the next **`agent`**, **`plan`**, or **`docs`** turn uses the updated **`models`**, **`tools`**, and other fields from the swapped in-memory config. **`mcp_servers`** changes apply fully to **new** sessions only.

### Session memory REST (**`-tags=http,memory`**)

These **`/foxxycode/sessions/{id}/memory/*`** routes register only when **`foxxycode`** is built with **`memory`** (**`external/httpserver/memory_http.go`**). A plain **`http`** build without **`memory`** returns **`404`** (**`memory_http_stub.go`**).

| Method | Path | Notes |
|--------|------|-------|
| GET | **`/foxxycode/sessions/{id}/memory/tree`** | Without **`root`**, lists **`global`** and **`workspace`**. Otherwise lists allowed **`.md` / `.txt`** children (traversal guarded). |
| GET | **`/foxxycode/sessions/{id}/memory/file`** | Query **`root`** + **`path`**. UTF-8 content. |
| PUT | **`/foxxycode/sessions/{id}/memory/file`** | JSON **`{"root","path","content"}`**. |
| POST | **`/foxxycode/sessions/{id}/memory/dir`** | JSON **`{"root","path"}`** for new subdirectory. |
| DELETE | **`/foxxycode/sessions/{id}/memory/file`** | Query **`root`** + **`path`**. Removes a **`.md` / `.txt`** file or deletes a directory tree recursively. **`400`** when **`path`** targets the memory root itself. |

### Scheduler REST (**`-tags=http,scheduler`**)

Paths are **missing** from a plain **`http`** build and from OpenAPI when **scheduler** is not linked (client **404**). When linked, responses use the same **`ErrorEnvelope`** JSON as other **`/foxxycode`** routes. **`503`** when **`scheduler.enabled`** is false for that process. Handlers plus merged scheduler OpenAPI fragments live in **`external/httpserver/scheduler_http.go`** (**`http`**,**`scheduler`**); **`scheduler_http_stub.go`** registers nothing when **scheduler** is omitted.

| Method | Path | Notes |
|--------|------|-------|
| GET | **`/foxxycode/scheduler/jobs`** | JSON **`{ scheduler, jobs[] }`**. Envelope **`scheduler`** includes **`enabled`**, resolved **`dir`**, **`timeout`**, **`max_queue`**, **`runs_active`**, **`retain_sessions`**. Optional query **`include_body=true`** embeds each job instruction body. Each **`jobs[].running`** is **true** only while this server process tracks an in-flight agent run for that job (not from a stray **`basename.lock`** alone). Stale locks older than a grace window are removed during list and daemon ticks. |
| POST | **`/foxxycode/scheduler/jobs`** | Create job. JSON body **`job_id`**, **`description`**, **`schedule`** (5-field UTC cron), optional **`paused`**, **`cwd`**, **`model`**, **`mode`**, **`body`**. **`201`** + **`Location`**. **`409`** when **`job_id`** already exists. |
| GET | **`/foxxycode/scheduler/jobs/{job_id}`** | Full **`SchedulerJob`** including **`body`**. |
| PUT | **`/foxxycode/scheduler/jobs/{job_id}`** | Replace entire job file. |
| PATCH | **`/foxxycode/scheduler/jobs/{job_id}`** | Merge fields (e.g. **`paused`**, **`schedule`**, **`body`**). Optional **`job_id`** in the body renames the on-disk job (moves **`.md`**, **`.state`**, **`.lock`** when idle). Response **`job_id`** is the effective id after the patch. **`409`** if the target id exists or the job is busy. |
| DELETE | **`/foxxycode/scheduler/jobs/{job_id}`** | Remove **`.md`** and sidecars **`basename.state`**, **`basename.lock`** when idle. **`409`** when locked or a run is tracked. |
| POST | **`/foxxycode/scheduler/jobs/{job_id}/pause`** | Sets YAML **`paused: true`**. |
| POST | **`/foxxycode/scheduler/jobs/{job_id}/resume`** | Sets **`paused: false`**. |
| POST | **`/foxxycode/scheduler/jobs/{job_id}/run`** | Fire-and-forget manual run (**`202`**). Does **not** advance cron **`.state`** (cron timing stays independent). **`409`** when paused, busy, or locked. |
| POST | **`/foxxycode/scheduler/jobs/{job_id}/cancel`** | **`context.Cancel`** on the active tracked run when possible. If nothing is tracked but an old **`basename.lock`** remains (e.g. after a crash), the server may remove that lock and still set **`cancelled`** to **true**. JSON **`{ cancelled: bool }`**. |
| GET | **`/foxxycode/scheduler/jobs/{job_id}/runs`** | Metadata for persisted runs (**`session_id`**, timestamps, **`status`**). **`limit`** query (default **50**, max **100**). Read full transcript with **`GET /foxxycode/sessions/{session_id}/messages`**. |

Process logs for scheduler should stay short; full traces live under **`sessions.dir`** in normal session layout with **`schedulerRun`** metadata.

## **`model`**, profiles, and direct completion

- **`POST /v1/chat/completions`** and **`POST /v1/responses`** require **`model`** to be exactly one **`id`** from **`GET /v1/models`** (session operating profiles **agent** / **plan** / **docs** or a **`models[].model`** entry).
- **agent** / **plan** / **docs** selects the FoxxyCode agent (ReAct, tools). Optional JSON **`metadata.model`** (**`models[].model`** selector) updates the session **`SelectedModelID`**. Omit **`metadata`** or omit **`metadata.model`** to follow **`EffectiveModelID`** (session **`selectedModelId`** then **`agent.model`**). **`metadata.model`** set to **`null`**, **`""`**, or an unknown selector is **400**. Optional **`metadata.reasoning`** sets the session reasoning level (resolved after any **`metadata.model`** change); it must be one of the effective model's **`reasoning_levels`** (**`null`** / **`""`** clears it; any other unsupported value is **400**). The agent maps the level to OpenAI **`reasoning_effort`** or Anthropic extended-thinking **`budget_tokens`**.
- **`model`** listing a **`models[].model`** YAML selector runs **one direct LLM request** per call (no ReAct); **`metadata`** with a **`model`** key on that branch is rejected with **400** (use **`model`** as the routing field only).

Non-stream replies include **`metadata`** with **`model`** set to the effective YAML backend selector (**`api_model`** may expose the REST model id).

## Sessions and headers

### Without **`X-FoxxyCode-Session-ID`**

`POST /v1/chat/completions` / **`POST /v1/responses`** allocate a random session bundle for that flow (ACP **`session/new`**). The bundle's working directory is the **current project** (**`GET /foxxycode/project`**) when one is set, else the process default cwd. An explicit **`-cwd`** flag / **`FOXXYCODE_CWD`** at startup seeds the current project; otherwise the last opened project is restored from `~/.foxxycode/projects.json`.

### With **`X-FoxxyCode-Session-ID`**

1. Active in-memory bundle wins.
2. Otherwise load from disk when **`session.json`** exists.
3. Otherwise pin **`session/new`** with the supplied identifier (fresh empty bundle folders).

Malformed ids (**HTTP 400**). Dedicated **`/foxxycode/*`** helpers return **503** if persistence is unavailable (`Manager` lacked a **`FileStore`**, primarily in tests).

### Per-session model (bundled UI)

- **`GET /foxxycode/sessions/{id}/messages`** returns **`model`** (effective YAML backend for the session), **`selectedModelId`** (stored override in **`session.json`**, may be empty), and **`selectedReasoning`** (effective reasoning level for the session, empty when the model has none).
- **`PATCH /foxxycode/sessions/{id}`** accepts **`selectedModelId`** to set the YAML **`models[].model`** selector for that session (unknown ids **400**) and **`selectedReasoning`** to set the reasoning level (must be one of the model's **`reasoning_levels`**; empty clears it; unsupported value **400**).
- The SPA restores **Model** from **`model`** when opening a chat. Changing **Model** writes cookie **`foxxycode_llm_model`** (default for the next **New chat**) and **`PATCH`**es the active session.

## Memory roots

Matches `external/memory/README.md`: **`global`** uses configured **`memory.dir`** (fallback **`$FOXXYCODE_HOME/memory`**). **`workspace`** resolves to **`$CWD/memory`** for that session bundle. **`agentMemory`** placeholders remain agent-only (`session.json`), not REST-editable here.

Interactive tool permission prompts are bypassed when **`tools.permission_mode`** is **`bypass`**. Otherwise the bundled UI shows **`event: permission`** during **`POST /v1/responses`** (**`stream: true`**) and completes the turn via **`POST /foxxycode/sessions/{id}/permission`** (same pattern as **`question`**).

## CLI flags

Equivalent to **`foxxycode acp`**: **`--config`**, **`--home`**, **`--cwd`**, **`--sessions-dir`**, **`--session-id`**, **`--log-*`**, optional **`--scheduler-enabled`**. Networking flags: **`-H` / `--host`**, **`-P` / `--port`**. Defaults fall back to YAML **`httpserver.host` / port** when left at **`0.0.0.0:12345`**.

## Build

```bash
# HTTP API only (no npm, no embedded SPA root)
make build TAGS=http

# HTTP API plus embedded SPA (**make ui-build** runs first)
make build TAGS="http ui"
```

`go test ./...` skips **`external/httpserver`** unless **`go test -tags=http`**. SPA-specific tests compile under **`go test -tags=http,ui`** (Makefile **`make test`** runs **`ui-build`** once, then **`http`** and **`http,ui`** and scheduler combinations).

For a manual gateway check against a disposable **`foxxycode http`** process, **`examples/test_httpserver.sh`** runs the Python demos under **`examples/httpserver/`** (see **`examples/README.md`**). Steps that call chat or responses expect a working **`models`** backend.
